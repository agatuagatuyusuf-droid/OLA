# OLA 全量功能 EXE 自动化搭建平台开发任务书

## 0. 项目一句话目标
我要开发一个自用的 Windows EXE 桌面程序，把 OLA 插件的全量能力封装成中文可视化动作库，让我可以马上开始搭建自动化流程，并且支持手动执行、定时执行、日志查看、失败截图、自检轮询，直到功能真正可用为止。

这个项目不是商业 SaaS，不做多用户，不做云平台，不做复杂权限。目标是我自己马上投入工作。

---

# 1. 最终产品形态
软件名称暂定：SkyAuto OLA 自动化助手

产品形态：一个 Windows EXE 桌面程序

核心要求：
1. 打开 EXE 后显示全中文 UI
2. 能创建多个自动化流程
3. 能把 OLA 的功能作为中文动作添加到流程里
4. 能保存流程
5. 能立即运行流程
6. 能设置定时运行流程
7. 能查看每次执行日志
8. 失败时自动截图
9. 内置自检/轮询机制，不通过就继续修
10. 不允许假装完成

---

# 2. 技术架构固定方案
必须使用以下技术路线，不要擅自换架构：
- 开发语言：C#
- 目标平台：Windows 10 / Windows 11
- UI：WPF
- 架构模式：MVVM
- 数据库：SQLite
- ORM：Dapper 或 EF Core，优先 Dapper
- 定时任务：Quartz.NET
- 配置文件：appsettings.json
- 流程保存格式：JSON
- 底层自动化：OLA 插件 DLL / COM / 官方调用方式
- 日志：SQLite + 本地 log 文件
- 截图：本地 screenshots 目录
- 素材库：本地 assets 目录

不要使用：Vue、FastAPI、Docker、Redis、WebSocket、多用户系统、商业授权系统、插件市场、拖拽画布作为第一版。

第一版必须做成本地 EXE 桌面软件，保证最快能用。

# 4. UI 页面要求
## 首页 Dashboard
- 今日执行次数、成功/失败次数
- 最近失败流程
- OLA 插件状态
- 定时任务状态
- 自检状态
- 按钮：新建自动化 / 立即自检 / 查看日志 / 打开 OLA 设置

## 自动化列表
- 流程名称、启用状态、步骤数量、是否有定时、最近执行时间/结果
- 操作：运行 / 编辑 / 复制 / 删除 / 日志
- 支持新建、复制、删除、搜索、启用/停用、立即运行

## 步骤编辑器
- 表格 + 参数面板
- 字段：顺序、动作分类、动作名称、参数摘要、超时、重试、启用/停用、操作
- 支持：添加/删除步骤、上移下移、编辑参数、保存流程、导入导出 JSON、立即运行

## 素材库
- 类型：图片模板、OCR 区域、找色点、文件、账号变量、普通变量
- 支持：上传、截图取样、命名、删除、预览、在步骤参数选择

## 定时任务
- 支持每天固定时间、间隔 N 分钟、每周指定、开机自动启动
- 显示：绑定流程、定时规则、下次执行时间、最近执行结果、启用状态

## 执行日志
- 显示：流程名称、开始/结束时间、总耗时、结果、失败步骤、错误信息、失败截图
- 详情显示每步：序号、动作名称、开始结束时间、耗时、状态、输入/输出、错误、截图

## OLA 设置
- 设置插件路径
- 测试连接
- 显示机器码/授权状态/日志/截图路径
- 一键打开数据目录

## 自检轮询
- 必检 30 项：数据库、配置、OLA 连接、流程增删改查、动作注册、步骤执行、截图、素材、定时、日志、UI
- 状态：未检查 / 检查中 / 通过 / 失败
- 显示失败原因和证据
- 按钮：开始自检 / 重新检查失败 / 导出报告

# 6. 流程 JSON 标准
每个流程保存如下结构，步骤包含动作类型、参数、超时、重试、启用状态

```json
{
  "id": "workflow_001",
  "name": "自动登录网站",
  "variables": { "账号": "admin", "密码": "123456" },
  "steps": [
    { "index":1, "category":"系统", "type":"open_program", "params":{"path":"chrome.exe"} },
    { "index":2, "category":"图像", "type":"wait_image", "params":{"image_asset_id":"login.png","timeout":15} },
    { "index":3, "category":"图像", "type":"click_image", "params":{"image_asset_id":"login.png","similarity":0.85} }
  ]
}
```

# 7. OLA 功能封装
必须封装核心动作，初版重点动作包括：系统类、窗口类、鼠标键盘、图像类、OCR类、文件类、网络类、日志类。  
- 系统类：打开程序、打开网址、CMD、sleep、关闭进程  
- 窗口类：查找窗口、绑定、解绑、激活、移动、截图  
- 鼠标键盘：移动、点击、双击、右键、滚轮、输入、按键、快捷键  
- 图像类：截图、找图、等待图、点击图、判断图、找色、多点找色  
- OCR类：OCR区域、OCR图片、识别数字、查找文字、判断文字  
- 文件类：读写文件、复制移动删除文件  
- 网络类：HTTP GET/POST、下载文件  
- 日志类：写入日志、保存截图  

必须设计 OlaFunctionRegistry 统一注册表，UI 所有动作都从这里读取，不能假装可用。

---

# 8. 流程执行器 & 步骤执行器
- WorkflowRunner 读取 JSON 按顺序执行步骤
- IActionStepExecutor 统一接口，每步返回 StepExecutionResult
- StepExecutionResult 包含成功/失败、输出、截图、异常
- 执行规则：失败重试、记录日志、失败截图、终止流程

---

# 9. 定时任务
- 使用 Quartz.NET
- 支持每天固定、间隔、每周、开机自动启动
- 触发后调用 WorkflowRunner.RunAsync
- 同一流程不允许重复执行

---

# 10. 自检轮询机制
- 必检 30 项 (数据库、配置、OLA、流程增删改查、动作注册、步骤执行、截图、素材、定时、日志、UI)
- 每轮自检必须返回状态 + 证据
- 状态：pending / running / pass / fail
- 失败必须修复，再轮询直到通过

---

2. 每轮必须运行自检
3. 必须贴出自检结果摘要
4. 有失败项必须继续修
5. 不允许把未实现功能标记为通过
6. 所有按钮必须能点击
7. 所有页面必须能打开
8. 所有数据库表必须真实创建
9. 日志必须真实写入 SQLite
10. 截图必须真实保存
11. 导入导出必须生成真实文件
